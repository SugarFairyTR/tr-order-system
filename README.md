# 🔥 주식회사 티알코리아 주문 자동화 시스템 (v3.0)

## 📋 프로젝트 개요

외근 업무 중 실시간 주문 처리를 위한 **Progressive Web App (PWA)** 기반 주문 자동화 시스템입니다.
**Firebase 클라우드 자동 저장**으로 저장 버튼 한 번만 클릭하면 모든 팀원과 실시간 데이터 공유가 가능합니다.

## 🚀 시작하기

### 1️⃣ **Firebase 클라우드 설정 (5분)**
```bash
🔥 Firebase 클라우드 자동 저장 설정 (권장!)
```

#### **빠른 설정**
1. 앱 설정 화면에서 **"🔥 Firebase 클라우드 자동저장 설정"** 버튼 클릭
2. **"🚀 Firebase 콘솔 열기"** 버튼으로 바로 이동
3. Google 계정으로 로그인 후 프로젝트 생성
4. Realtime Database 활성화
5. 웹앱 설정 후 config 복사
6. `firebase-config.json` 파일 생성 및 업로드

#### **상세 설정 가이드**
앱에서 **"📖 단계별 가이드"** 버튼을 클릭하면 자세한 설정 방법을 확인할 수 있습니다.

### 2️⃣ **기본 파일 설정**
```bash
# 모든 파일을 웹 서버 디렉토리에 복사
# (예: Apache htdocs, Nginx html 등)
```

### 3️⃣ **사용자 설정**
`user_config.json`에서 담당자 추가/수정:
```json
{
  "users": {
    "홍길동": {
      "pin": "1234",
      "name": "홍길동",
      "role": "영업사원",
      "email": "hong@company.com"
    }
  }
}
```

### 4️⃣ **데이터베이스 설정**
- `database_optimized.json`: 최신 최적화된 데이터
- `database_converted.json`: 레거시 지원용

### 5️⃣ **웹 서버 실행**
```bash
# Python 개발 서버 (테스트용)
python -m http.server 8000

# 또는 Node.js 서버
npx serve .
```

### 6️⃣ **브라우저에서 접속**
```
http://localhost:8000
```

## 🎯 사용 방법

### 🔑 **로그인**
1. 담당자 선택
2. 4자리 PIN 입력
3. 로그인 완료

### 📝 **주문 입력 ( 자동화!)**
1. **담당자** → **판매처** → **도착지** 순서로 선택
2. **분류** → **품목** 선택
3. **수량, 단가, 도착일, 도착시간** 입력
4. **💾 저장 버튼 한 번만 클릭!**
5. **"✅ 주문이 클라우드에 저장되었습니다!"** 메시지 확인
6. **끝! 더 이상 파일 관리 필요 없음**

### 🌐 **내근직 실시간 확인**
1. [Firebase 콘솔](https://console.firebase.google.com/) 접속
2. 프로젝트 → Realtime Database → orders
3. **모든 영업사원의 주문을 실시간으로 확인**
4. JSON 내보내기로 엑셀 변환 가능

### 🔍 **주문 검색**
- 검색창에 키워드 입력 (주문번호, 담당자, 판매처 등)
- 담당자/날짜 필터 적용
- 과거 주문 표시 옵션 활용

### 📊 **통계 확인**
- 주문 목록 화면에서 **통계** 버튼 클릭
- 실시간 매출 및 주문 분석 확인

### ✏️ **주문 수정/삭제**
1. **수정** 탭에서 주문 선택
2. **수정하기** 또는 **삭제** 실행
3. Firebase에도 자동 반영

## 🆘 문제 해결

### **Firebase 관련 문제**
```bash
# Firebase 연결 안됨
1. firebase-config.json 파일 확인
2. 프로젝트 ID와 설정 값 재확인
3. 네트워크 연결 상태 확인

# 데이터 동기화 안됨
1. 브라우저 새로고침
2. 네트워크 연결 확인
3. Firebase 콘솔에서 데이터 확인
```

### **일반적인 문제**
```bash
# 데이터가 표시되지 않는 경우
1. F12 → Console 탭에서 오류 확인
2. 페이지 새로고침 (Ctrl+F5)
3. 브라우저 캐시 삭제

# 오프라인에서 작동하지 않는 경우
1. HTTPS 연결 확인
2. Service Worker 등록 확인
3. 브라우저 저장 공간 확인
```

### **데이터 복구**
```javascript
// 브라우저 Console에서 실행
// Firebase 동기화 재시도
app.syncWithFirebase();

// 수동 데이터 복구
app.repairData();

// 데이터 무결성 검사
app.validateDataIntegrity();
```
